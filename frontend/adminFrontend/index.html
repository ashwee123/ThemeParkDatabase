System configuration

  • Park hours & seasons: visitor_park + special events tables — manage via SQL or future PATCH bundle.
  • Ticket pricing tiers: visitor portal constants + ticket; centralize in config service.
  • Backups / restore: use managed MySQL snapshots (RDS, PlanetScale, etc.); document RPO/RTO here for ops.
  • Notifications: route notificationlog + email/SMS webhooks per environment.
  • Branding: per-portal CSS and messaging in each frontend; promote shared design tokens when refactoring.

Security & compliance

GDPR / CCPA: honor data export & deletion for visitors through the visitor API (hash stripping, ticket anonymization). Retention policies belong in legal config + scheduled jobs.
